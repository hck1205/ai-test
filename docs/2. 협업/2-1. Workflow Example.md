# 2-1. Workflow Example

## Architecture Overview

UI는 디자이너가 **Atomic Design Pattern**을 기반으로 AI를 활용하여 생성하고,  
개발자는 **Feature-Sliced Design(FSD)** 패턴을 기반으로 기능을 구현하는 구조로 개발을 진행합니다.

이 구조는 **UI 설계와 기능 개발을 명확히 분리**하여 AI 기반 개발 환경에서 코드 안정성과 확장성을 확보하는 것을 목표로 합니다.

Designer

- Atomic UI Component 생성
- Design System 유지
- UI 일관성 관리

Developer

- Feature 중심 기능 개발
- 상태 관리 및 비즈니스 로직 구현
- 시스템 구조 관리

---

# Key Development Principles

### Spec First

코드를 작성하기 전에 반드시 **기능 Spec을 먼저 정의합니다.**

Spec에는 다음 정보가 포함됩니다.

- 기능 목적
- 사용자 행동 흐름
- UI 요구사항
- 상태 변화
- 예외 상황

Spec을 먼저 정의하면

- AI가 구현해야 할 목표가 명확해지고
- 코드 구조 일관성이 유지되며
- Prompt 반복을 줄일 수 있습니다.

---

### Small Scope Generation

AI에게 큰 기능을 한 번에 요청하지 않고 **작은 단위로 구현합니다.**

개발 흐름

Feature 정의  
↓  
Component 구조 정의  
↓  
파일 단위 코드 생성

이 방식의 장점

- 코드 안정성 증가
- AI 수정 범위 최소화
- 예측 가능한 코드 생성

---

### Validation

코드 생성 이후 **Spec과 구현이 일치하는지 검증합니다.**

검증 항목

- Spec에 정의된 기능 구현 여부
- UI 동작 일치 여부
- 상태 흐름 정상 동작 여부

---

### Regression Protection

새 기능 개발 시 기존 기능이 깨지지 않도록 **테스트 기반 검증을 수행합니다.**

주요 테스트

- Unit Test
- Component Test
- E2E Test

이를 통해 AI 코드 수정으로 인한 기능 손상을 방지합니다.

# Design Layers

## Spec

기능 요구사항을 정의합니다.

Spec은 **사용자 행동과 기능 목적 중심**으로 작성됩니다.

예

Feature: Login

User Flow

1. User enters email
2. User enters password
3. User clicks login

Behavior

- validate email format
- disable button while loading
- show error on failure
- redirect on success

---

## State Model

Feature에서 사용하는 상태 구조를 정의합니다.

예

State: login

email: string  
password: string  
loading: boolean  
error: string | null

---

## UI Tree

UI Tree는 화면의 **UI 계층 구조**를 정의합니다.

예

```
LoginPage
 ├ Logo
 ├ LoginForm
 │   ├ EmailInput
 │   ├ PasswordInput
 │   └ SubmitButton
 └ ErrorMessage
```

---

## Component Graph

UI Tree를 기반으로 **컴포넌트 관계를 정의합니다.**

예

LoginPage  
↓  
LoginForm  
↓  
EmailInput  
PasswordInput  
SubmitButton

이 단계는 컴포넌트 의존 관계를 명확히 합니다.

---

## Dependency Graph

코드 의존 관계를 정의합니다.

예

login.api  
↓  
useLogin  
↓  
LoginForm  
↓  
LoginPage

AI는 이 구조를 기반으로 코드 생성 순서를 결정합니다.

---

## Task Graph

구현 작업 단위를 정의합니다.

예

1 create login api  
2 create login state  
3 create useLogin hook  
4 create LoginForm component  
5 connect state to UI  
6 add validation

Task Graph는 **AI 작업 순서를 정의하는 실행 계획입니다.**

---

# Patch Workflow

AI는 항상 **작은 patch 단위로 작업합니다.**

예

create useLogin.ts  
create login.api.ts  
update LoginForm.tsx

이 방식은 다음 문제를 줄입니다.

- Prompt Ping-Pong
- 코드 구조 붕괴
- 대규모 수정
