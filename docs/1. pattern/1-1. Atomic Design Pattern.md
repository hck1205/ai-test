# 1-1. Atomic Design Pattern

## 개념

Atomic Design은 UI를 가장 작은 단위부터 조합하여 설계하는 방식입니다.  
주로 디자인 시스템 구축과 UI 재사용성을 높이기 위해 사용됩니다.

UI를 원자 단위부터 점점 큰 단위로 조합합니다.

---

## 구조

### Atoms (원자)

UI의 가장 작은 단위입니다.

예

- Button
- Input
- Label
- Icon
- Text

특징

- 더 이상 쪼갤 수 없는 최소 UI
- 스타일과 기본 동작만 존재
- 비즈니스 로직 없음

### Molecules (분자)

여러 개의 Atom을 조합한 작은 UI 구성입니다.

예

- SearchInput  
  (Input + Button)  

- FormField  
  (Label + Input + ErrorMessage)  


특징

- 작은 기능 단위
- 재사용 가능한 UI 묶음

---

### Organisms (유기체)

여러 Molecule과 Atom이 모여 만든 **큰 UI 영역**입니다.

예

- Navbar
- ProductCard
- Header
- CommentList

특징

- 하나의 독립적인 UI 블록
- 페이지의 특정 영역을 담당

---

### Templates (템플릿)

페이지의 **레이아웃 구조**를 정의합니다.

예

- ProductLayout
- DashboardLayout
- BlogLayout

특징

- Header / Sidebar / Content 배치
- 데이터 없음
- 구조만 존재

---

### Pages (페이지)

실제 데이터가 들어간 **최종 화면**입니다.

예

- ProductPage
- LoginPage
- DashboardPage

특징

- API 데이터 사용
- 실제 사용자 화면

---

    molecules/
      SearchInput

      ProductPage

## 장점

- UI 재사용성이 높음
- 디자인 시스템 구축에 적합
- 디자이너와 협업이 쉬움
- Storybook과 궁합이 좋음

## 단점

- 비즈니스 로직 위치가 애매함
- 기능 중심 개발과 맞지 않을 수 있음
- 프로젝트 규모가 커질수록 의존성 관리가 어려움
- 도메인 맥락이 약함
