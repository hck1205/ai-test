# 1-2. Feature-Sliced Design (FSD)

Feature-Sliced Design은 **기능(Feature)과 도메인 중심으로 코드를 구조화하는 프론트엔드 아키텍처 패턴**입니다.

Atomic Design이 **UI 구조 중심**이라면  
FSD는 **비즈니스 로직과 기능 중심 구조**입니다.

대규모 서비스에서 **유지보수성과 확장성**을 높이기 위해 사용됩니다.

---

app

---

# FSD 레이어 구조

| 레이어    | 한국어 의미   | 역할                    |
| --------- | ------------- | ----------------------- |
| App       | 애플리케이션  | 전체 앱 설정            |
| Processes | 비즈니스 흐름 | 여러 기능이 연결된 흐름 |
| Pages     | 페이지        | 라우트 단위 화면        |
| Widgets   | 위젯          | 페이지 구성 UI 블록     |
| Features  | 기능          | 사용자 행동 단위 기능   |
| Entities  | 도메인 모델   | 핵심 데이터 모델        |
| Shared    | 공통          | 공통 UI 및 유틸         |

---

# Shared (공통 레이어)

## 의미: 공통 모듈

## 역할

프로젝트 어디에서나 사용할 수 있는 코드

## 예시

#### UI 컴포넌트

- Button
- Input
- Modal

#### 유틸

- date utils
- formatting

#### API

- http client

#### 예시 구조

src/shared/

ui/
Button
Input
Modal

lib/
date
format

api/
axiosClient

---

# Entities (도메인 모델)

## 의미: 엔티티 / 도메인 객체

## 역할

서비스에서 사용하는 **핵심 데이터 모델**

- User
- Product
- Order

## 예시

Product

- product type
- product card UI
- product api

## 예시 구조

src/entities/product/

ui/
ProductCard

model/
product.types.ts

api/
productApi.ts

---

# Features (기능)

## 의미: 기능

## 역할

사용자가 수행하는 **행동 단위 기능**

예

login  
add-to-cart  
comment

## 예시

AddToCart

- 버튼 클릭
- cart api 호출
- 상태 관리

## 예시 구조

src/features/add-to-cart/

ui/
AddToCartButton

model/
useAddToCart.ts

api/
cartApi.ts

# Widgets (위젯) = Entity + Feature + UI Composition

## 의미: 페이지 구성 블록

## 역할

여러 **Feature**와 **Entity**를 조합한 **큰 UI 구성**

예

ProductList  
Header  
Sidebar

## 예시 구조

ui/
ProductList.tsx

useProductList.ts

# Processes (비즈니스 흐름)

## 의미: 비즈니스 프로세스

## 역할

예
