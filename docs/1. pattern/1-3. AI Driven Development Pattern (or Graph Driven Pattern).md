# 1-3. AI Driven Development Pattern (or Graph Driven Pattern)

# Architecture 구조

AI 기반 개발에서는 다음 설계 단계를 거칩니다.

Spec  
↓  
State Model  
↓  
UI Tree  
↓  
Component Graph  
↓  
Dependency Graph  
↓  
Task Graph  
↓  
Patch  
↓  
Code  
↓  
Validation  
↓  
Tests

---

# Spec (기능 명세)

기능의 요구사항을 정의합니다.

Spec은 **사용자 행동과 기능 목적 중심**으로 작성합니다.

예

Feature: Login

User Flow

1. User enters email
2. User enters password
3. User clicks login

Behavior

- validate email format
- disable button while loading
- show error on failure
- redirect on success

---

# State Model (상태 모델)

Feature에서 사용하는 상태 구조를 정의합니다.

예

State: login

email: string  
password: string  
loading: boolean  
error: string | null

이 단계는 상태 구조를 명확히 하여 AI 코드 생성 오류를 줄입니다.

---

# UI Tree (UI 구조)

UI Tree는 **화면의 계층 구조**를 정의합니다.

예

LoginPage

```
├ Logo
 ├ LoginForm
 │   ├ EmailInput
 │   ├ PasswordInput
 │   └ SubmitButton
 └ ErrorMessage
```

이 단계는 **UI 구조와 컴포넌트 관계를 정의**합니다.

---

# Component Graph (컴포넌트 관계)

UI Tree를 실제 컴포넌트 관계로 표현합니다.

예

LoginPage  
↓  
LoginForm  
↓  
EmailInput  
PasswordInput  
SubmitButton

이 단계는 **컴포넌트 의존 관계를 명확하게 합니다.**

---

# Dependency Graph (코드 의존성)

컴포넌트뿐 아니라 **코드 의존 관계**까지 정의합니다.

예

login.api  
↓  
useLogin  
↓  
LoginForm  
↓  
LoginPage

AI는 이 그래프를 기반으로 코드 생성 순서를 이해합니다.

---

# Task Graph (작업 단위)

구현 작업 단위를 정의합니다.

예

1 create login api  
2 create login state  
3 create useLogin hook  
4 create LoginForm component  
5 connect state to UI  
6 add validation

AI는 이 Task Graph를 기반으로 작업을 수행합니다.

---

# Patch Workflow

AI는 항상 **작은 patch 단위로 작업합니다.**

예

create useLogin.ts  
create login.api.ts  
update LoginForm.tsx

이 방식은 다음 문제를 줄입니다.

- Prompt Ping-Pong
- 코드 구조 붕괴
- 대규모 수정

---

# Tests (테스트)

Regression 방지를 위해 테스트를 작성합니다.

테스트 레이어

Component Test  
Feature Test  
E2E Test

예

Button.test.ts  
LoginForm.test.ts  
login.e2e.test.ts

---

# Designer와 Developer 역할

shared/ui

Input

---

Feature 중심 기능 구현

features  
widgets

# 전체 개발 흐름

Spec 작성

↓  
UI Tree 정의  
↓  
Component Graph 정의  
↓  
Dependency Graph 정의  
↓  
Task Graph 생성  
↓  
Patch 단위 코드 생성  
↓  
Validation  
↓  
Tests
